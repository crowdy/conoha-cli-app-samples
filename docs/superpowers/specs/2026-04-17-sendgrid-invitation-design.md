# SendGrid Invitation Sample App Design

## Overview

組織の管理者がメンバー候補に招待メールを送るシンプルな Web アプリ。
Next.js (フロントエンド) + FastAPI (バックエンド) + nginx (Basic Auth リバースプロキシ) の3コンテナ構成。

## Directory Structure

```
sendgrid-invitation/
├── compose.yml
├── .env.server          # SENDGRID_API_KEY, FROM_EMAIL, FROM_NAME
├── nginx/
│   ├── nginx.conf       # リバースプロキシ + Basic Auth
│   └── .htpasswd        # 認証ファイル（README 手順で生成）
├── frontend/
│   ├── Dockerfile
│   ├── package.json
│   └── src/             # Next.js アプリ（招待フォーム）
├── backend/
│   ├── Dockerfile
│   ├── requirements.txt
│   └── main.py          # FastAPI（SendGrid メール送信）
└── README.md
```

## Container Architecture

```
[Browser] → :80 [nginx (Basic Auth)] → :3000 [Next.js frontend]
                                      → :8000 [FastAPI backend]
```

### Services

| Service  | Image             | Port         | Role                          |
|----------|-------------------|--------------|-------------------------------|
| nginx    | nginx:alpine      | 80 (public)  | Basic Auth + reverse proxy    |
| frontend | node:20-alpine    | 3000 (internal) | Invitation form UI         |
| backend  | python:3.12-slim  | 8000 (internal) | `/api/invite` endpoint, SendGrid |

### Routing

- `/api/*` → FastAPI backend
- All other paths → Next.js frontend

## Environment Variables (`.env.server`)

| Variable          | Description            |
|-------------------|------------------------|
| `SENDGRID_API_KEY` | SendGrid API key      |
| `FROM_EMAIL`       | Sender email address  |
| `FROM_NAME`        | Sender name (org name)|

## API

### `POST /api/invite`

**Request body:**

```json
{
  "to_email": "member@example.com",
  "to_name": "田中太郎",
  "message": "チームに参加してください"
}
```

**Fields:**

| Field      | Type   | Required | Description          |
|------------|--------|----------|----------------------|
| `to_email` | string | yes      | Recipient email      |
| `to_name`  | string | no       | Recipient name       |
| `message`  | string | no       | Invitation message   |

**Response (success):**

```json
{ "success": true }
```

**Response (error):**

```json
{ "detail": "error message" }
```

- Validation via Pydantic model (email format check on `to_email`)
- Uses `sendgrid` Python package with `Mail` object
- Returns appropriate HTTP status codes on error

## Frontend

- Single page (`/`)
- Form fields: recipient email (required), recipient name (optional), message body (optional)
- Success/error message displayed after submission
- Plain `fetch` to `/api/invite`, no state management library needed

## Email Content (Plain Text)

```
{from_name} からの招待

{to_name} 様

{from_name} があなたをメンバーとして招待しています。

{message}

---
このメールは {from_name} から送信されました。
```

## Basic Auth

- nginx enforces Basic Auth on all routes
- `.htpasswd` file generated by user via `htpasswd` command (documented in README)
- Credentials not committed to repository

## What This Sample Does NOT Include

- Database / persistence
- External auth services (Clerk, etc.)
- HTML email templates
- Auto-reply / confirmation emails
- SendGrid Dynamic Templates
